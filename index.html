🧩 Sistemas y Plataformas

SIATEP
Sistema encargado de gestionar expedientes (CUN), generar documentos (RECA, RELA) y ejecutar procesos de transmisión hacia la SUSESO mediante SISESAT.
SISESAT
Sistema Nacional de Información de Seguridad y Salud en el Trabajo. Plataforma administrada por la SUSESO que centraliza la información relacionada con la Ley 16.744.
SUSESO
Superintendencia de Seguridad Social, organismo regulador que recibe y valida la información.
IST
Instituto de Seguridad del Trabajo. Cliente que genera información de accidentes.

📄 Conceptos de Negocio

RECA (Resolución de Calificación)
Clasifica el accidente:
  • RECA 2 → Accidente de trayecto
  • RECA 4 → Accidente del trabajo
  • RECA 10 → Dirigente sindical
RELA
Documento que informa reposos laborales y se transmite a SUSESO.
CUN
Código Único Nacional que identifica el expediente.
Atención
Identificador del evento médico asociado al paciente.

🗄️ Base de Datos

MED_SIATEP_RECA
Tabla donde se registran las RECA.
MED_SIATEP_RELA
Tabla donde se registran los RELA y su estado de transmisión.
ADH_MEDICOS
Tabla que valida médicos permitidos para transmisión.
Logs de envío
Registros que confirman si un documento fue transmitido.

🔄 Procesos Clave

Flujo de Transmisión
  1. Generación de documento (RECA / RELA)
  2. Registro en tablas SIATEP
  3. Proceso batch interno
  4. Envío a SISESAT
  5. Confirmación en logs
Regularizar Documentos
Permite reprocesar documentos desde la interfaz SIATEP:
SIATEP → Resumen Envío SUSESO → Regularizar Documentos
Detalle Envío SUSESO
Vista que muestra el estado de los envíos realizados.

🧑‍💻 Gestión de Tickets

RF
Ticket de requerimiento (ej: RF-6130897).
REQ
Identificador interno del requerimiento.
Pendiente Cliente
Estado que indica espera de validación del cliente.
Evidencia
Pruebas de ejecución o corrección.

⚠️ Problemas Típicos

Caso frecuente:
  • Documento generado ✅
  • Registrado en base de datos ✅
  • No aparece en logs ❌
  • No se transmite ❌
Posibles causas:
  • Filtros en procesos (ej: NOT IN / condiciones)
  • Datos inválidos (ej: médico no permitido)
  • Errores en procesos batch
  • Problemas en UI
  • Datos incompletos

🧪 Queries SQL de Diagnóstico SIATEP

1. Buscar RECA por CUN 
SQL> SELECT *
  2  FROM MED_SIATEP_RECA
  3  WHERE CUN = :CUN;
2. Validar estado de envío 
SQL> SELECT CUN, TIPO_RECA, IND_ENVIO_SUSESO
  2  FROM MED_SIATEP_RECA
  3  WHERE CUN = :CUN;
3. Revisar RELA asociados 
SQL> SELECT *
  2  FROM MED_SIATEP_RELA
  3  WHERE CUN = :CUN;
4. Validar médico 
SQL> SELECT *
  2  FROM ADH_MEDICOS
  3  WHERE RUT_MEDICO = :RUT;
5. Diagnóstico completo 
SQL> SELECT
  2      R.CUN,
  3      R.TIPO_RECA,
  4      R.IND_ENVIO_SUSESO,
  5      R.RUT_MEDICO,
  6      M.RUT_MEDICO AS MEDICO_VALIDO,
  7      CASE
  8          WHEN M.RUT_MEDICO IS NULL THEN 'MEDICO NO VALIDO'
  9          WHEN R.IND_ENVIO_SUSESO = 'N' THEN 'NO TRANSMITIDO'
 10          WHEN R.IND_ENVIO_SUSESO = 'S' THEN 'OK'
 11          ELSE 'REVISAR'
 12      END AS DIAGNOSTICO
 13  FROM MED_SIATEP_RECA R
 14  LEFT JOIN ADH_MEDICOS M
 15      ON R.RUT_MEDICO = M.RUT_MEDICO
 16  WHERE R.CUN = :CUN;

💡 Tip real

Esto es oro

El 80% de los problemas SIATEP van a caer en:

  • ❌ Médico no válido
  • ❌ No pasó por proceso batch
  • ❌ Quedó en BD pero no transmitido
